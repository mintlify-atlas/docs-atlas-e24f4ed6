## Overview

`SimpleCommandGroup` allows you to organize multiple related subcommands under a single main command. For example, `/arena join`, `/arena leave`, `/arena list` would all be subcommands of the `/arena` command group.

The command group automatically handles help menus, tab completion, and subcommand routing.

## Creating a command group

### Constructors

<ParamField path="SimpleCommandGroup()" type="constructor">
  Creates a new command group using `SimpleSettings.MAIN_COMMAND_ALIASES`.
  
  Requires a settings class extending SimpleSettings with `Command_Aliases` defined.
</ParamField>

<ParamField path="SimpleCommandGroup(String labelAndAliases)" type="constructor">
  Creates a new command group with label and aliases separated by `|` or `/`.
  
  **Example:** `"arena|a|ar"` creates `/arena` with `/a` and `/ar` as aliases.
  
  <Expandable title="parameters">
    <ParamField path="labelAndAliases" type="String" required>
      The main label and aliases separated by `|` or `/`
    </ParamField>
  </Expandable>
</ParamField>

<ParamField path="SimpleCommandGroup(String label, List<String> aliases)" type="constructor">
  Creates a new command group with explicit label and aliases.
  
  <Expandable title="parameters">
    <ParamField path="label" type="String" required>
      The main command label
    </ParamField>
    <ParamField path="aliases" type="List<String>" required>
      List of command aliases
    </ParamField>
  </Expandable>
</ParamField>

<ParamField path="SimpleCommandGroup(StrictList<String> labelAndAliases)" type="constructor">
  Creates a new command group from a list where the first item is the label and the rest are aliases.
  
  <Expandable title="parameters">
    <ParamField path="labelAndAliases" type="StrictList<String>" required>
      List containing label and aliases
    </ParamField>
  </Expandable>
</ParamField>

### Basic example

```java
public class ArenaCommandGroup extends SimpleCommandGroup {
    
    public ArenaCommandGroup() {
        super("arena|a|ar");
    }
    
    @Override
    protected void registerSubcommands() {
        registerSubcommand(new ArenaJoinCommand());
        registerSubcommand(new ArenaLeaveCommand());
        registerSubcommand(new ArenaListCommand());
    }
}
```

## Registration

### register()

<ParamField path="register()" type="void">
  Registers this command group into Bukkit and starts using it.
  
  Automatically registers all subcommands added in `registerSubcommands()`.
</ParamField>

### unregister()

<ParamField path="unregister()" type="void">
  Removes this command group from Bukkit.
  
  Takes immediate effect in the game.
</ParamField>

### isRegistered()

<ParamField path="isRegistered()" type="boolean">
  Checks if the command group has been registered.
  
  <Expandable title="returns">
    <ParamField path="boolean" type="boolean">
      True if registered
    </ParamField>
  </Expandable>
</ParamField>

```java
ArenaCommandGroup arenaCmd = new ArenaCommandGroup();
arenaCmd.register();

// Later...
if (arenaCmd.isRegistered()) {
    arenaCmd.unregister();
}
```

## Core methods

### registerSubcommands()

<ParamField path="registerSubcommands()" type="abstract void">
  **Required implementation.** Override this method to register your subcommands.
  
  Call `registerSubcommand()` for each subcommand you want to add.
</ParamField>

```java
@Override
protected void registerSubcommands() {
    registerSubcommand(new CreateSubCommand());
    registerSubcommand(new DeleteSubCommand());
    registerSubcommand(new ListSubCommand());
    
    // Add help text
    registerHelpLine(
        "&6Arena Commands:",
        "&e  Manage your arena battles"
    );
}
```

## Registering subcommands

### registerSubcommand(SimpleSubCommand command)

<ParamField path="registerSubcommand(SimpleSubCommand command)" type="void">
  Registers a new subcommand for this group.
  
  <Expandable title="parameters">
    <ParamField path="command" type="SimpleSubCommand" required>
      The subcommand instance to register
    </ParamField>
  </Expandable>
</ParamField>

### registerSubcommand(Class\<? extends SimpleSubCommand\> parentClass)

<ParamField path="registerSubcommand(Class<? extends SimpleSubCommand> parentClass)" type="void">
  Automatically registers all extending classes for the given parent class.
  
  Ignores abstract classes. Child classes must be final.
  
  <Expandable title="parameters">
    <ParamField path="parentClass" type="Class<? extends SimpleSubCommand>" required>
      The parent subcommand class
    </ParamField>
  </Expandable>
</ParamField>

```java
@Override
protected void registerSubcommands() {
    // Auto-register all subcommands extending ArenaSubCommand
    registerSubcommand(ArenaSubCommand.class);
}
```

<Note>
When using auto-registration, ensure all child classes are `final` to avoid accidental registration of intermediate classes.
</Note>

### registerHelpLine(String... menuHelp)

<ParamField path="registerHelpLine(String... menuHelp)" type="void">

  Displayed in the help menu along with subcommands.

  </Expandable>
</ParamField>


## Complete example

```java
public class ArenaCommandGroup extends SimpleCommandGroup {
    
    public ArenaCommandGroup() {
        super("arena|a");
    }
    
    @Override
    protected void registerSubcommands() {
        // Register individual subcommands
        registerSubcommand(new ArenaJoinCommand());
        registerSubcommand(new ArenaLeaveCommand());
        registerSubcommand(new ArenaListCommand());
        registerSubcommand(new ArenaCreateCommand());
        registerSubcommand(new ArenaDeleteCommand());
        
        // Add custom help text
        registerHelpLine(
            "",
            "&6Arena System:",
            "&7  Create and manage PvP arenas",
            "&7  Players can join, leave, and battle"
        );
    }
    
    @Override
    protected boolean sendHelpIfNoArgs() {
        return true; // Show help when /arena is typed
    }
    
    @Override
    protected ChatColor getTheme() {
        return ChatColor.RED; // Use red theme for arena commands
    }
    
    @Override
    protected String getCredits() {
        return "&7Arena plugin by &fYourName";
    }
    
    @Override
    protected String[] getHelpHeader() {
        return new String[] {
            "&8&m------------------------------",
            "&c&lArena Commands &7v1.0",
            " ",
            "&2  [] &f= Optional parameter",
            "&c  <> &f= Required parameter",
            " "
        };
    }
}
```

## Automatic features

The command group automatically provides:

<Tabs>
  <Tab title="Help menu">
    Players can type `/arena help` or `/arena ?` to see all subcommands they have permission for.
    
    The help menu includes:
    - Command usage syntax
    - Description (on hover in 1.7+)
    - Required permission (on hover)
    - Click to suggest command
    - Automatic pagination
  </Tab>
  
  <Tab title="Tab completion">
    Automatically completes:
    - Subcommand names when typing `/arena <tab>`
    - Delegates to subcommand tab completion for further arguments
    - Only shows subcommands the player has permission for
  </Tab>
  
  <Tab title="Permission handling">
    - Subcommands inherit the main command label
    - Default permission format: `plugin.command.{label}.{sublabel}`
    - Players only see subcommands they can use
  </Tab>
  
  <Tab title="Error handling">
    - Invalid subcommands show an error
    - No permissions show appropriate message
    - Automatic usage display on invalid arguments
  </Tab>
</Tabs>

## Registration patterns

### In your main plugin class

```java
public class MyPlugin extends SimplePlugin {
    
    @Override
    protected void onPluginStart() {
        ArenaCommandGroup arenaCmd = new ArenaCommandGroup();
        arenaCmd.register();
    }
}
```

### Using auto-register annotation

```java
@AutoRegister
public class ArenaCommandGroup extends SimpleCommandGroup {
    // Constructor and methods...
}
```

<Note>
The `@AutoRegister` annotation automatically registers your command group when the plugin starts, as long as it has a no-args constructor.
</Note>
